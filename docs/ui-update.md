# UI Update

## 수정된 레이아웃 구조

- 제조 화면은 `prompt / game-stage / progress`의 3단 구조로 재편했다.
- `game-stage`는 CSS Grid 기반으로 구성했다.
- 중앙에는 항아리 오브젝트, 좌우에는 각각 향료 버튼 5개씩 배치했다.
- 향료 버튼은 absolute positioning 대신 컬럼 내부의 flex 흐름으로 정렬된다.
- 뷰포트가 줄어들면 버튼과 중앙 오브젝트가 함께 축소되고, 좌우 위치 관계는 유지된다.

## 사운드 이벤트

- hover: `/public/sounds/glass-clink.mp3`
- click: `/public/sounds/water-pouring.mp3`
- 향수 완성: `/public/sounds/magic-wand.mp3`
- 사운드는 HTML5 Audio API로 미리 로드하고 버튼 이벤트에 연결했다.
- 마지막 향 선택 시 마법 사운드를 재생한 뒤 결과 화면으로 이동한다.

## LLM 프롬프트 변경

- 결과 설명 생성은 OpenAI 호출 전용 route handler를 통해 처리한다.
- 프롬프트는 아래 방향으로 변경했다.

```text
너는 향수 공방에서 향수를 설명하는 감성적인 카피라이터다.

사용자가 아래 문장에 맞춰 향수를 만들었다.

문장:
[사용자가 받은 문장]

향 레시피:
[선택된 향 비율]

이 향들이 섞였을 때 느껴질 수 있는 분위기와 감정을 상상하여
아름다운 시적인 문장으로 설명해라.

조건
- 한국어
- 줄바꿈 포함
- 너무 길지 않게
- 감정적인 표현 사용
```

## 반응형 레이아웃 방식

- 좌우 향료 버튼은 각 컬럼의 `max-width`와 버튼 `width: 100%` 조합으로 반응형 축소된다.
- 상단 문장 박스는 dark glass overlay로 바꿔서 어두운 배경 위에서도 가독성을 확보했다.
- 중앙 항아리와 진행도 패널은 별도 column 안에서 정렬되어, 창 크기가 바뀌어도 스테이지 구도가 유지된다.
- 진행도 바에는 애니메이션 그라데이션을 적용했다.

## Codex에게 마지막 한 문장

현재 화면을 웹폼 UI가 아니라 작은 웹 게임처럼 보이도록 인터랙션과 연출을 강화해줘.
